# Excalidraw diagrams to create

This file tracks ASCII-art placeholders in `/docs/` that should be upgraded to
real Excalidraw drawings. It lives at the top level of `/docs/` and starts with
`_` so the `sync-docs.ts` script skips it.

## Workflow

1. Open the target page in production (e.g. `eduskript.org/<admin-slug>/manual/writing-content/...`)
2. Create the diagram with the in-page Excalidraw toolbar button — Eduskript generates both light and dark SVG variants automatically
3. Name the file per the "Suggested filename" column below
4. Edit the corresponding `.md` file: remove the HTML comment + ASCII code fence, replace with the `![caption](filename.excalidraw)` line (already provided in each placeholder comment)
5. Re-run `npx tsx scripts/sync-docs.ts` to propagate to the seeded docs

## Where the placeholders live

Each placeholder is an HTML comment directly above an ASCII art code block.
Search for `EXCALIDRAW PLACEHOLDER` in the codebase to find them all:

```bash
grep -rn "EXCALIDRAW PLACEHOLDER" docs/
```

The HTML comments are invisible to students viewing the rendered page — but
they serve as clear markers for anyone editing the source.

## Style guidelines

All diagrams should:
- Use Excalidraw's default hand-drawn style (don't switch fonts)
- Stay readable when rendered at ~600px wide (docs default container)
- Have both light and dark SVG variants (auto-generated by Eduskript's
  in-page editor if you draw there)
- Use short labels that stay legible at small size
- Avoid tiny text (< 14pt Excalidraw font size)
